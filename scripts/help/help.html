ZeroGrid help

You can find below useful help information about the remote session, it's abilities, and other useful things.

Downloaded AppImage application fails to start

AppImage applications has FUSE dependency (required program for it to start). Even if that dependency is met (installed), FUSE needs systemd to run (system management service). Since remote desktops are containerized, systemd can't run in the because of security considerations (systemd is a system wide daemon, not just any program).

Solution: instead of running AppImages directly, you need to extract them (as they are like a archives). Firstly, open directory where AppImage is located (images shown below are from file manager, but for terminal same steps apply), and then on empty space in directory make right click, and select option Open Terminal Here.



After that, enter the name of the executable in the terminal with prefix ./ and after executable name type --appimage-extract with space between name and the suffix. Press Enter after.



Shortly after, new folder will appear in the same directory as executable, usually called squashfs-root. Make sure that you don't have same directory in that folder, or it will be overwritten! Open it, and in the same directory, there will be application called AppRun. Open it with double-click and your AppImage will be open and ready to go.



Tip: if you don't feel typing the whole executable name, you can type beginning of the name, and then after pressing Tab, computer will automatically fill the rest.

Changing the password and password-related issues

By default, password of root and user isn't set. You can change that by entering in the terminal sudo passwd <user that you want to change password for>. For local user, you can instead use sudo passwd.
Certain programs like su won't work until you set a password for user.
Also note that the sudo executable (to execute commands as superuser) doesn't ask for password by default. This introduces big security issues if password isn't set! View here for more information on how to do that.

Package installation

Distribution comes initially without updated packages. To install any of them, you first need to update the package repository. Open the terminal, and in it type sudo apt update. After that, you can install packages (see them on the web to find it's names) using sudo apt install <package name>.

Package installation (for packages downloaded from the web)

To install package from internet, while downloading choose option with .deb (you can see below an download example).



After that, open directory in file manager where application downloaded, and again, as above, open that folder in the terminal. After that, you can type sudo dpkg -i <downloaded package/file>. Package will be installed.



If installation sometimes failed with error like asking for more dependencies, try entering in the terminal sudo apt install --fix-broken after updating packages (as above).

Snap or Flatpak package installation fail

As said above (AppImage section), systemd isn't supported. You need to resort to some other options, like package installation or using AppImages.
